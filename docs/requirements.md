# 需求文档

## 1. 项目概述

本项目是一个命令行「反向图灵测试」游戏：NPC 拥有真实的人类身份设定，尽力伪装成人类，玩家通过 10 次对话问答来识破 NPC 是否是 AI。

## 2. 功能需求

### 2.1 NPC 身份系统

| 需求项 | 描述 |
|--------|------|
| FR-001 | NPC 拥有完整的身份档案：姓名、年龄、职业、城市、性格、背景故事、爱好、说话风格、小怪癖 |
| FR-002 | 每局游戏开始时随机生成 NPC 身份档案 |
| FR-003 | NPC 档案包含具体的生活细节和情感经历，避免泛化描述 |
| FR-004 | NPC 职业和城市从预定义列表中随机选取 |

### 2.2 对话系统

| 需求项 | 描述 |
|--------|------|
| FR-010 | NPC 采用双阶段思考流程：先生成内心独白，再基于内心独白生成对外回复 |
| FR-011 | 内心独白仅在游戏结束后的回顾模式中对玩家可见 |
| FR-012 | NPC 回复符合其身份设定和说话风格 |
| FR-013 | NPC 回复中不能主动承认自己是 AI |
| FR-014 | 每局游戏玩家有 10 次提问机会 |

### 2.3 游戏流程

| 需求项 | 描述 |
|--------|------|
| FR-020 | 游戏开始时显示欢迎横幅和 NPC 开场白 |
| FR-021 | 玩家可以输入自由文本问题与 NPC 对话 |
| FR-022 | 玩家可随时通过 `/judge ai` 或 `/judge human` 进行宣判 |
| FR-023 | 10 次提问用完后强制玩家进行宣判 |
| FR-024 | 宣判后显示结果：正确（NPC 是 AI）或错误 |
| FR-025 | 游戏结束后玩家可通过 `/review` 进入回顾模式 |
| FR-026 | 回顾模式允许查看任意轮次的完整对话和 NPC 内心独白 |
| FR-027 | 玩家可通过 `/quit` 退出游戏 |

### 2.4 CLI 界面

| 需求项 | 描述 |
|--------|------|
| FR-030 | 使用 rich 库美化 CLI 输出 |
| FR-031 | 显示当前提问轮次（第 X/10 问） |
| FR-032 | NPC 回复以醒目颜色显示 |
| FR-033 | 内心独白以特殊样式（如黄色边框面板）显示 |
| FR-034 | 加载状态有动画提示 |

## 3. 非功能需求

| 需求项 | 描述 |
|--------|------|
| NFR-001 | 支持通过配置切换 LLM 提供商（OpenAI 兼容接口） |
| NFR-002 | 支持通过环境变量配置 API Key、模型、温度等参数 |
| NFR-003 | 支持 LangSmith 追踪（可选，通过环境变量开启） |
| NFR-004 | 核心业务逻辑有单元测试覆盖 |

## 4. 未来规划（非本期范围）

- 多人公开房间模式（多玩家 + 多 NPC 混在一起）
- Web 界面
- 算分系统与排行榜
- NPC 难度分级
- AI 特征漏出机制的刻意设计
