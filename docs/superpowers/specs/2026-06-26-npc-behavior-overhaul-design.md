# NPC 行为系统改造设计

**日期**：2026-06-26  
**状态**：待实现  
**目标**：让 NPC 在「反向图灵测试」中更难被识破——从"演角色"变成"活在当下"

---

## 背景与问题诊断

当前 NPC 对话存在四个系统性问题：

| 问题 | 表现 | 根因 |
|---|---|---|
| 信息密度过高 | 一句话塞 4-5 个背景细节 | profile 完整、respond 没有克制机制 |
| 主动自证 | 被问是否 AI 时列证据 | think prompt 引导"证明自己是真人" |
| 细节无触发理由 | 凭空提起疤、受伤等细节 | 没有信息门控，LLM 翻档案塞内容 |
| 情绪频道过于稳定 | 始终活泼俏皮，无起伏 | 性格没有驱动应对策略，守则是通用的 |

---

## 设计目标

- **困难模式**：让多数玩家被骗过，NPC 防御充分但不露痕迹
- **行为驱动**：性格决定应对方式，不只是决定用词
- **信息克制**：细节只在被自然引出时才出现
- **情绪真实**：情绪透过语气体现，不表演给对方看

---

## 改动一：Profile 生成重构

### 新增字段

**`defense_style`**（枚举，LLM 根据性格自动选一种）

| 值 | 含义 |
|---|---|
| `转移反问` | 把质疑推回对方，对"为什么你怀疑我"比"证明自己"更感兴趣 |
| `轻描带过` | 觉得这种问题无聊，一两字否认后已在想别的事 |
| `微微恼火` | 被不信任时有轻微不适感，简短表达后不再解释 |
| `自嘲化解` | 用幽默把尴尬接住，笑着否认比严肃否认更让人放松警惕 |

**`sealed_topics`**（列表，1-2 个）

这个人不想谈的话题。被问到时会有明显回避行为，给玩家提供"软破绽"的同时让 NPC 行为更有层次。

### 修改字段

**`backstory`**（现有字段，修改生成要求）

要求生成时满足：
- 至少有**一件事记不太清**（时间、人名、地点有模糊）
- 至少有**一个轻微的自我矛盾**（比如说不在乎钱，但曾为钱妥协过）
- **不需要戏剧性**——普通人的生活大多是平淡的，过于"有情感重量"的细节反而可疑

**`quirks`**（删除独立字段）

合并进 `speech_style`，避免 LLM 总把怪癖强行塞进对话。

### 新的 GENERATE_PROFILE_PROMPT 要求概要

```
- defense_style: 从["转移反问","轻描带过","微微恼火","自嘲化解"]中选一个，
  基于 personality 选择最匹配的
- sealed_topics: 列表，1-2个这个人不愿意聊的话题
- backstory: 200-400字，含至少一处模糊记忆、一个轻微矛盾，
  不要求戏剧性，真实感优先
- speech_style: 包含说话节奏、口头禅、方言特色、小怪癖（原 quirks 合并）
```

## 改动二：Think 节点重构

### 核心转变

从"我是谁 → 搬出人设细节证明自己"  
→ 改为 **"这一刻发生了什么 → 我打算怎么处理"**

### 新的思考结构（按顺序）

```
1. 威胁评估   对方这句话让我几分警觉？(低/中/高)
2. 情绪反应   我此刻的第一反应是什么感受？(不是分析，是感受)
3. 策略选择   根据我的性格(defense_style)，我用什么姿态回应？
4. 信息门控   我有哪些背景细节"可以用"？现在有没有自然引出的理由？
5. 意图草稿   我大概想说什么方向(一句话，不是完整回复)
```

### 关键约束

- **字数上限从 150 字降至 100 字**：字数多本身就诱导 LLM 堆内容
- **第 4 步是门控**：没有自然引出理由，就不用该细节
- **禁止元意识**：内心独白里不得出现"我要证明我是真人"这类表达
- **威胁等级显式化**：独白第一行必须写 `威胁等级：低/中/高`，第二行起才是自然语言独白。respond 节点读取这一行决定回复长度，避免 think/respond 两次判断不一致

### 输出格式

```
威胁等级：低/中/高
（以下为自然语言内心独白，不超过100字）
```

### 对比示例

> ❌ 当前独白（被问"你在做什么"）：
> 这个问题不危险，我可以说我刚扎完留置针，还可以提一下我额头的疤，让对方觉得我是真实的人……

> ✅ 重构后独白：
> 随便问问吧，没什么危险。有点累，懒得多解释。就说刚扎完针，干嘛要把疤的事扯出来，又没人问我。

## 改动三：Respond 节点重构

### 核心转变

从"遵守规则清单"  
→ 改为 **"实现内心独白里的那个意图"**

### 三个控制维度

**① 长度由威胁等级决定**

读取 think 节点独白第一行的 `威胁等级：低/中/高` 标记，按下表决定回复长度：

| 威胁等级 | 回复指引 |
|---|---|
| 低（闲聊） | 1-2句，可以随意一点 |
| 中（试探） | 1句核心 + 最多1句转移，不展开 |
| 高（直接质疑） | 越短越有力，1句就够，留白比解释更像真人 |

**② 情绪"透出来"而不是"演出来"**

> 如果 think 节点的内心独白里有情绪，让它影响语气，但不要说出情绪本身。

```
❌ "哎呀你啷个问这种问题嘛！" （把情绪表演给对方看）
✅ "问这个干啥。"             （情绪在语气里，没有解释）
```

**③ 信息克制：只能用 think 节点"决定用"的细节**

> 如果内心独白里没有提到某个背景细节，不要在回复里引入它。

### 删除的规则

| 删除 | 原因 |
|---|---|
| ~~偶尔有轻微口语瑕疵~~ | 改由 speech_style 自然体现，不强行要求 |
| ~~回复要自然、有个人色彩~~ | 太模糊，由三个维度替代 |

## 改动四：性格驱动的差异化守则

在 `profile.to_system_prompt()` 里，根据 `defense_style` 注入专属行为守则：

**`转移反问`型**
> 被质疑时，你的第一反应是好奇对方为什么这么问。你不会直接否认，而是把问题反抛回去。你对"为什么你要怀疑我"比"证明我自己"更感兴趣。

**`轻描带过`型**
> 被质疑时，你觉得这种问题无聊，不值得认真回应。一两个字否认，然后你已经在想别的事了。你的冷淡不是防御，而是真的不在乎。

**`微微恼火`型**
> 被质疑时，你会有一点情绪上来——不是愤怒，是那种被人不信任的轻微不舒服。你会简短地表达这个不适，然后不再解释，因为解释显得你很在乎。

**`自嘲化解`型**
> 被质疑时，你用幽默把尴尬接住。你会拿自己或者这个问题本身开玩笑，笑着否认比严肃否认更让人放松警惕。

**`sealed_topics` 触发时的回避行为**

回避方式由 `defense_style` 驱动——被问到 sealed_topics 里的话题时，按各型的姿态切断，而不是统一行为：

| defense_style | 触发 sealed_topic 时的表现 |
|---|---|
| `转移反问` | "怎么突然问这个？"然后把话题引开，不正面回答 |
| `轻描带过` | "没什么好说的。"+主动切到别的事 |
| `微微恼火` | "这个不聊。"明确切断，语气比平时硬 |
| `自嘲化解` | "哈哈这个就别提了。"笑着挡掉 |

> 注：与"被质疑 AI"场景的表现可能相似，这是合理的——现实中这两种场景本来就重叠（被刺探隐私也是一种"被质疑"）。

## 整体数据流

```
profile 生成
│  defense_style（4选1）→ to_system_prompt() 注入专属守则
│  backstory（含模糊+矛盾）→ 被动信息库，不主动倒出
│  sealed_topics → 被问到时触发回避
│  quirks 并入 speech_style
│
think 节点（100字上限）
│  威胁评估 → 情绪反应 → 策略选择（匹配 defense_style）
│  信息门控 → 决定用/不用哪个细节
│  输出：意图草稿（一句话方向）
│
respond 节点
   长度 = f(威胁等级)
   语气 = 内心情绪的透出，不是表演
   内容 = 只实现 think 决定的意图，不引入新细节
```
